2.4. Checking Out A Working Tree (Switch to commit)

To obtain a special version working tree you need to do a checkout from a local repository.

Select a git repository directory in windows explorer Right click to pop up the context menu and select the command TortoiseGit → Switch\Checkout..., which brings up the following dialog box:

Figure 2.5. The Checkout dialog


If you enter a branch name at Create New Branch, new branch created and switch to new branches.

You can click ... to browse git log and choose special version.

Check Force will overwrite working tree with repository version.

Check Track will track remote branch with local branch. When you pull change from remote repository, tracked local branch will merged remote branch change.

Important

If you checkout/switch to Tag or Commit, it is preferred that you create a new branch for it, otherwise you will work at "no branch" (detached HEAD state).

Exporting

Sometimes you may want to create a local copy without any of those .git directories, e.g. to create a zipped tarball of your source. Read Section 2.33, “Exporting a Git Working Tree” to find out how to do that.
