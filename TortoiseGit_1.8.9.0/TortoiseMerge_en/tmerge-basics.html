TortoiseGitMerge has several different uses:

TGM有以下几种用途：

  • Viewing the difference between two files and merging changes into one of them, or removing unwanted changes.

    审查两个文件的不同和整合修改点到一个文件中，或者移除掉不想要的改动点。

  • Editing conflicts between local changes and changes in the Git repository following an update.

    编辑本地文件同Git仓库里的更新版本之间的改动冲突。

  • Applying patch files. This is the reason TortoiseGitMerge was created :)

    应用补丁代码。这也是为什么TGM会存在的原因。

2.1. Viewing and Merging Differences  审查并合并改动点

In this mode you are comparing two files. The file in the left pane is considered to be the original file (sometimes referred to as Theirs, and the file in the right pane is the modified file (sometimes referred to as Mine.

在这种模式下，你要对比两个文件。左边面板的是原来的文件（有时候表示别人的改动），右边面板是修改过的文件（有时候指向我自己的改动）。

You can make simple line-based changes to the file in the right pane which include:

你可以对右边面板的文件基于行进行修改：

  • Reverting changed lines back to the text shown in the left pane.

    将改动行回复到与左边面板显示的内容一致的状态

  • Using a combination of both blocks, either Theirs before Mine or Mine before Theirs.

    使用组合块，要么是“他们的改动点在我的改动点前面”，要么是“我的改动点在他们的改动点前面”

You can also edit the file in the right pane just as you would in a text editor. Such lines are marked using a pencil icon. Please note that if you want to make any of the line/block-based changes described above, it is better to do those first since once you start editing the file yourself it becomes impossible for TortoiseGitMerge to keep track of the relationship to the original files.

你也可以在右边面板中编辑文件，就像你在文件编辑器中编辑文件一样。这样的行被标记了铅笔图案。请记住，如果你想进行如上所述的基于行或者基于块的改动，你最好在编辑文件之前进行，以免使得TGM无法追踪文件修改关系。
